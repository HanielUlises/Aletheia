# Aletheia — DEL-based Epistemic Planner

**IεPC 2026 submission · Team 3**

Aletheia is a C++17 epistemic planner for the International Epistemic Planning Competition (IεPC) at ICAPS 2026. It implements search over Kripke models using Dynamic Epistemic Logic (DEL) product update with bisimulation contraction, targeting the **Basic** and **Intermediate** tracks of the competition.

The planner supports both linear and conditional epistemic planning over S5ₙ and KD45ₙ models, including public announcements, private ontic actions, semi-private sensing, common knowledge goals, and knowing-whether (`Kw`) reasoning.

# Features

## Supported

* S5ₙ epistemic frames
* KD45ₙ doxastic frames
* Public announcement actions
* Semi-private sensing actions
* Private ontic actions
* Modal preconditions/effects/goals
* Common knowledge formulas
* Knowing-whether (`Kw`) formulas
* Multi-pointed epistemic states
* Conditional AND-OR planning
* Bisimulation contraction
* Greedy and heuristic search
* Grounded universal formulas
* Disjunctive formulas
* Negative formulas and effects

## Not Supported

* Lists
* List comprehensions
* Existential quantification
* General frame classes beyond S5/KD45
* Event-condition formulas
* Universal/disjunctive list formulas

# Architecture

The planner is a single self-contained C++17 binary with no runtime dependencies beyond:

* C++ standard library
* `nlohmann/json`

The input is a grounded JSON planning task generated by `plank`, and the output is a JSON plan.

---

# Core Modules

| Module           | Description                            |
| ---------------- | -------------------------------------- |
| `parser`         | Loads grounded JSON planning tasks     |
| `formula`        | Recursive modal formula representation |
| `state`          | Kripke model representation            |
| `product_update` | DEL product update implementation      |
| `bisimulation`   | Canonical model minimization           |
| `action`         | Applicability and event semantics      |
| `heuristic`      | Heuristic evaluation functions         |
| `search`         | GBFS, EHC, and AO* planners            |
| `validator`      | Internal plan validator                |

---

# Search Strategies

Strategy selection is automatic but may also be forced via command-line flags.

| Strategy | Usage                                        |
| -------- | -------------------------------------------- |
| AO*      | Conditional epistemic planning with sensing  |
| EHC      | Small shallow tasks                          |
| GBFS     | Large or effectively linear epistemic spaces |

---

# Heuristics

| Flag | Name               | Description                       |
| ---- | ------------------ | --------------------------------- |
| `ug` | Unsatisfied Goals  | Counts unsatisfied goal conjuncts |
| `ed` | Epistemic Distance | Measures accessibility violations |
| `ks` | Knowledge Spread   | Optimized for `Kw`-heavy tasks    |
| `wc` | World Count        | Number of designated worlds       |

# Benchmark Summary

## Automatic Strategy Selector

The planner automatically selects between:

* **AO*** for conditional epistemic planning
* **EHC** for shallow deterministic tasks
* **GBFS** for large or effectively linear state spaces

Selection is based on:

```math
\text{avg\_branch}^{\text{depth}} \times \text{worlds} \times \text{agents} \times \text{kd45\_discount}
```

where KD45 tasks receive a complexity discount due to seriality repair reducing effective branching.

# Observations

* `ed` and `ks` consistently outperform `ug` and `wc` on epistemically difficult domains.
* `wc` performs poorly on large branching domains because it provides weak guidance.
* `ks` is particularly effective on `Kw`-heavy communication domains such as Gossip.
* The automatic strategy selector correctly routes:

  * sensing-heavy tasks to AO*
  * shallow deterministic tasks to EHC
  * large effectively linear spaces to GBFS
* Bisimulation contraction significantly reduces AO* search explosion in Muddy Children and Coin domains.

# Known Issues from benchmarks

## amc1

The exported benchmark is missing the initial public announcement action ("at least one of you is muddy") that the classic Muddy Children solution requires. Without it, no child can ever deduce their own muddiness the ask_ChildN actions cycle without making progress because their event preconditions ([Kw. ?i] and <Kw. ?i>) depend on knowledge that cannot be established from the initial state alone. The planner correctly identifies this as unsolvable and terminates quickly. The domain file needs to be extended with the missing announcement action.

## cn5

The exported initial Kripke model makes the goal unreachable because announcement actions cannot alter propositional valuations.

The goal [B][A] has_B_n4 is unsatisfiable given the exported initial Kripke model. In the designated world w1, agent A's accessibility relation includes w2 where has_B_n4 is false, so [A] has_B_n4 fails at w1. Since w1 is B-accessible from w0, [B][A] has_B_n4 fails at w0. Announcement actions can only restrict accessibility relations — they cannot change which facts are true in worlds — so no sequence of announcements can make has_B_n4 true in w2. The goal is therefore unreachable by construction, and the planner correctly exhausts the reachable state space and returns null.
